## Findings

### TD815-I1 — ISSUE
**File:** `DECISIONS.md` (ADR-28, Decision item (2) and Evidence)
**Problem:** ADR-28 cites the sibling quote query as `portfolio/[ticker]/page.tsx:63-70`, but
at HEAD that block spans **:63-72**. Line 70 is `enabled: mounted && !!ticker,` — so the cited
range stops one line short of `refetchInterval: 30000` (line 69 within the block, file line 71),
which is *the exact line the citation exists to evidence*. ADR-28's central TD-15 claim is that
`refetchInterval` is deliberately not copied; the range it points a reader to excludes it.

This is the same defect class as LCB-I2 in the previous review (a citation that does not resolve
to the claimed construct). Its practical impact is low — the ADR is otherwise accurate and every
other citation resolves — but it is worth correcting precisely because this is the recurring
failure mode. Notably `AGENT.md:24` cites the same block correctly as `:64-72`, and the inline
comment at `page.tsx:65` also says `:64-72`, so ADR-28 is the sole outlier and is internally
inconsistent with the two other places describing the same code.

**Recommendation:** In `DECISIONS.md`, change both occurrences of
`portfolio/[ticker]/page.tsx:63-70` in ADR-28 to `portfolio/[ticker]/page.tsx:63-72` (or
`:64-72` to match AGENT.md and the inline comment — either resolves, but pick one and use it
consistently). Re-resolve the citation against the file after the edit.

For completeness, all other ADR-28 citations were verified line-by-line at HEAD and **do**
resolve: the 6 route citations (`chart:4,11-12`, `quote:3,10-11`, `search:2,8-9`, `news:3,12-13`,

### TD815-S1 — SUGGESTION
**File:** `app/(dashboard)/research/[symbol]/page.tsx:64-72` (the inline comment); `DECISIONS.md` ADR-28
**Problem:** The inline comment and ADR-28 both explain that declining `refetchInterval` means
the research page "does not poll." That is true only when the research page is mounted alone.
Because the two pages share one cache entry, if the position-detail page is mounted
simultaneously (or its observer is otherwise active), *its* 30s poll refreshes the shared
`["quote", symbol]` entry, and the research page — as a subscriber to that entry — will re-render
with the newer data. The research page never *initiates* polling, but it can still receive polled
updates. This is harmless (fresher data on a read-only surface) and is not a bug, but the docs
currently read as an absolute guarantee of no updates, which a future reader could rely on
incorrectly.

**Recommendation:** Soften the wording in the inline comment and ADR-28 from "this page does not
poll" to something like "this page never *initiates* polling; if the sibling page is mounted
concurrently its 30s poll refreshes the shared cache entry and this page will observe the update."
One sentence; no code change.

### TD815-S2 — SUGGESTION
**File:** `app/(dashboard)/research/[symbol]/page.tsx:61`; `app/(dashboard)/portfolio/[ticker]/page.tsx:49`
**Problem:** The shared cache hit depends on the two route params being byte-identical, but
neither page normalizes: research uses `params.symbol as string` and the sibling uses
`params.ticker as string`, both raw. `/research/aapl` and `/portfolio/AAPL` would therefore
produce `["quote","aapl"]` and `["quote","AAPL"]` — two entries, silently losing the cross-page
cache benefit that motivated the key sharing. In practice all in-app links are generated from
stored uppercase tickers so this does not misbehave today, and a case difference degrades to a
duplicate fetch rather than wrong data. Worth noting only because the plan's M1 manual check
would pass while a hand-typed lowercase URL quietly bypasses the optimization.

**Recommendation:** Optional and low priority. If the owner wants the sharing to be robust,
normalize at the key site on both pages — e.g. `queryKey: ["quote", symbol.toUpperCase()]` (and
the same on the sibling) — keeping the fetch URL as-is. Alternatively add a one-line note to the
AGENT.md fragile-surface entry that key sharing assumes canonical uppercase tickers. Do not
change one page without the other, or the sharing breaks entirely.
